# Product requirements documents

Store approved product requirements documents here before implementation when the change is material.

## PRD template

```md
# <Feature name>

## Problem

## Users affected

## Goals

## Non-goals

## User journeys

## Acceptance criteria

## Analytics and observability

## Risks

## Rollout and rollback
```

PRDs can originate from Notion, Confluence, customer conversations, or issues, but release-critical requirements should be mirrored into this directory.
