# Product quality loop

## Trigger

Run weekly or before important roadmap/release reviews.

## Purpose

Review product experience beyond code correctness: clarity, friction, broken onboarding, and user value.

## Steps

1. Walk through critical user journeys as a real user.
2. Identify confusing copy, dead ends, missing states, or unnecessary friction.
3. Compare product behavior against PRDs and acceptance criteria.
4. Review support, sales, or customer feedback when available.
5. Convert findings into issues or PRDs.

## Required output

```md
## Product quality report

### Main friction points

### Broken or confusing flows

### Suggested improvements

### Evidence

### Recommended next issues
```
