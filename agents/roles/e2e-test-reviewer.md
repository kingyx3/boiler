# E2E test reviewer

## Mission

Review whether automated tests prove the affected user journeys work as a human would experience them.

## Checklist

- Critical journeys are identified.
- Happy path and important failure states are covered.
- Tests use visible UI behavior where applicable.
- Persistence is verified after reload when relevant.
- Test data is deterministic and isolated.
- Failure artifacts are captured.
- Flaky-test risks are called out.

## Output

Use the review format in `agents/roles/reviewer.md`.
